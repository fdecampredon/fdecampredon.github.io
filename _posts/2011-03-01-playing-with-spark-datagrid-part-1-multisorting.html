Spark DataGrid is one of the best desktop component of the new flex hero, for thoses who struggled in the past trying to extend the old mx DataGrid or AdvancedDataGrid (like me), they will be happy to have finally a spark component easy to extend or to skin replacing it.

However Spark DataGrid is mainly a replacement for DataGrid, not AdvancedDatagrid, and so some features will miss to use the Spark DataGrid in some projects.

The good thing with spark DataGrid is about the Spark Architecture. By separating skin and logic, spark components are really easier to extend than the old mx components, I decided to try to implement some of this features on top of the new Spark DataGrid.

The first one is MultiSorting, and here is the result : MultiSortDatagrid

That DataGrid will apply visuals indicators for the initial sorting set on the dataProvider, track changes on the sort property of the dataProvider and apply visuals indicators, and allow user to use multisorting.

You can find the source for the MultiSort DataGrid here.

note : that component is not tested and can contain bugs
